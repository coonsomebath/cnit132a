Google Analytics 4 Analyzing User Traffic

Initial settings you may want to configure

Data retention

Admin > Data collection and modification > Data retention

There are two types: Event data and User data.

The default is 2 Months for Event data and 14 Months for User data. The maximum duration is 14 Months. Data Retention only affects explorations and funnel reports.

Internal traffic

Admin > Data collection and modification > Data streams > Stream Name (your target property) > Google tag > Configure tag settings > Settings > Show more > Define internal traffic

After you create the filter, you need to activate it. Write down the Data filter name and Parameter value for the next step.

Admin > Data collection and modification > Data filters > Create data filter

In Filter details, enter the Data filter name and Parameter value that you created in the previous step. In the Filter state section, select the Active radio button then press the Create button.

You may want to filter out localhost traffic if it is captured. If you have a static IP address, you may want to filter that out as well.

List unwanted referrals

This is relevant if you have links to external services, such as a payment gateway, that may redirect the user back to your site. If the user returns in such a way that technically counts as a new session, you may not want this counted as a referral.

Configure your domains

This is relevant if you have multiple domains hyperlinked/woven into your property that you want tracked as part of your user's journey.

Explore user traffic

Types of explorations

  • Path exploration
    • Visualize user interest by how they traverse through your property.
  • Funnel exploration
    • Visualize user "order of operation" for completed tasks.
    • May suggest ways to optimize user experience.
  • User Explorer
    • Select a group of users then focus on a particular user to observe their journey.
    • This may allow you to personalize, optimize, or troubleshoot user experience.
  • Free-form exploration
    • Create powerful custom analysis using modular building blocks.
  • Cohort exploration
    • Group users with common characteristics and observe their behavior over time.
  • e-Commerce exploration
    • Insightful analysis that allows you to observe points of interests like Checkout funnel performance and Promotion performance.
  • Segment overlap
    • Visualize how segments of users interrelate, intersect, etc.
    • You can use this insight as a basis for cohort formation.
  • User lifetime
    • Analyze user behavior over the lifetime of their interactions with your property.
    • Gain insight into points of interest like when was the last time your monthly active users made a purchase or was engaged with your property.
    • Find users with the highest lifetime value, with respect to revenues, and see which campaign was most effective for this cohort.

Free-form exploration technique

Benefits

  • Visualize data in a plot or tables
  • Freedom to choose row and column data to analyze
  • Ability to juxtapose multiple metrics
  • Ability to analyze nested group-by data
  • Ability to use filters and segmentation for granular insight
  • Create segments and audiences from your data

Funnel exploration technique

Benefits

  • Allows you to define which events pave the way to your business objective.
  • Allows you to quantitatively observe the performance of each of your defined event in terms of Completion and Abandonment rates.
  • Provides insight that may allow you to optimize user experience.
    • This may be the kind of analysis Amazon had conducted that led to the One-click shopping user experience innovation.
    • If your abandonment rate is higher than what you would like during the intermediary events that lead to your business objective, why not splice those out.
  • You can use Funnel exploration to measure the results of your UI/UX A/B testing experiments.
  • Depending on your how you define a quality prospect, you can choose to allocate some of your advertising budget to reengage abandoning visitors with Google Ads, etc.

Path exploration technique

Benefits

  • Analyze the URL paths that users take through your property.
    • For example, starting from the landing page or any page, you can see where users tend to go next depicted in a tree plot.
  • Gain insight into which pages users gravitate toward.
    • This may suggest what users really want from your property.
    • For example, if you have a product page but users tend to go to your blog, this may suggest that your product offering is not aligned with the user and you may want to change your product mix or drop your products page altogether and focus on the blog.
  • Retrace the users' path that ended with a particular event.
    • For example, if you end with the errors event, you can retrace the pathway that the user took to identify the page that has a hyperlink that sent the user a 404 Not Found error. You can then fix this bug.
    • Or you can end with a purchase event or some other conversion and retrace the steps that the user took.
      • With this additional insight, you can make informed decisions as to how to optimize site traffic.
